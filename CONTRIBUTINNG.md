# Contributing to React Hooks Playground
# مشارکت در React Hooks Playground

---

## 🇬🇧 English

Welcome! React Hooks Playground is an open educational project and we love contributions from the community — whether it's fixing a typo, adding a new Hook example, improving docs, or suggesting ideas.

### Ways to Contribute

- Add a new React Hook example
- Improve or fix existing examples
- Fix bugs or typos
- Improve documentation
- Suggest new features via Issues
- Help translate content

### Getting Started

1. **Fork** the repository
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/your-username/React-Hooks-Playground.git
   cd React-Hooks-Playground
   ```
3. **Install dependencies:**
   ```bash
   npm install
   ```
4. **Create a new branch:**
   ```bash
   git checkout -b feat/add-useContext-example
   ```
5. **Make your changes**, then run locally:
   ```bash
   npm run dev
   ```
6. **Commit** with a clear message:
   ```bash
   git commit -m "feat: add useContext example with theme toggle"
   ```
7. **Push** and open a **Pull Request** to the `main` branch.

### Branch Naming

| Type | Format | Example |
|------|--------|---------|
| New example | `feat/hook-name-example` | `feat/useReducer-example` |
| Bug fix | `fix/description` | `fix/useEffect-cleanup` |
| Docs | `docs/description` | `docs/update-readme` |

### Commit Message Format

```
feat: add useCallback example
fix: correct useEffect dependency array
docs: update contributing guide
style: fix typo in useState example
```

### Code Style

- Use functional components only
- Each Hook example should be self-contained in its own file
- Add clear comments explaining what the Hook does
- Keep examples simple and beginner-friendly

### Reporting Issues

Before opening an Issue, please search existing issues to avoid duplicates. Use the Issue templates provided.

## 🇮🇷 فارسی

به پروژه React Hooks Playground خوش آمدید! این یک پروژه آموزشی متن‌باز است و از مشارکت همه استقبال می‌کنیم — چه رفع یک اشتباه تایپی، چه اضافه کردن مثال جدید یا بهبود مستندات.

### روش‌های مشارکت

- اضافه کردن مثال جدید برای یک React Hook
- بهبود یا اصلاح مثال‌های موجود
- رفع باگ یا اشتباهات تایپی
- بهبود مستندات
- پیشنهاد ویژگی جدید از طریق Issue
- کمک در ترجمه محتوا

### مراحل شروع

1. ریپو را **Fork** کنید
2. نسخه fork شده را **Clone** کنید:
   ```bash
   git clone https://github.com/your-username/React-Hooks-Playground.git
   cd React-Hooks-Playground
   ```
3. **نصب پکیج‌ها:**
   ```bash
   npm install
   ```
4. یک **branch جدید** بسازید:
   ```bash
   git checkout -b feat/add-useContext-example
   ```
5. تغییرات را اعمال کنید و **لوکال تست کنید:**
   ```bash
   npm run dev
   ```
6. با یک پیام واضح **Commit** کنید:
   ```bash
   git commit -m "feat: add useContext example with theme toggle"
   ```
7. **Push** کنید و یک **Pull Request** به branch اصلی (`main`) باز کنید.

### نام‌گذاری Branch

| نوع | فرمت | مثال |
|-----|------|------|
| مثال جدید | `feat/hook-name-example` | `feat/useReducer-example` |
| رفع باگ | `fix/description` | `fix/useEffect-cleanup` |
| مستندات | `docs/description` | `docs/update-readme` |

### فرمت پیام Commit

```
feat: اضافه کردن مثال useCallback
fix: اصلاح dependency array در useEffect
docs: به‌روزرسانی راهنمای مشارکت
style: رفع اشتباه تایپی در مثال useState
```

### استانداردهای کد

- فقط از Functional Component استفاده کنید
- هر مثال Hook باید در فایل مجزا و مستقل باشد
- توضیح واضح در کامنت‌ها درباره عملکرد Hook بنویسید
- مثال‌ها باید ساده و مناسب برای مبتدیان باشند

### گزارش مشکل (Issue)

قبل از باز کردن Issue، ابتدا Issues موجود را جستجو کنید تا تکراری نباشد. از تمپلیت‌های Issue استفاده کنید.
